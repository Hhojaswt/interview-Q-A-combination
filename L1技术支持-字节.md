- # 1.  如何设置防火墙：

**Linux: 使用 iptables (通用性强)**

查看规则：iptables -L -n

允许 SSH (22端口)：iptables -A INPUT -p tcp --dport 22 -j ACCEPT

允许 Web 服务 (80/443)：iptables -A INPUT -p tcp --dport 80 -j ACCEPT

拒绝所有其他入站：iptables -A INPUT -j REJECT

注意：配置完规则后务必保存（如 service iptables save），否则重启后失效。

**使用 UFW (Ubuntu 推荐)**

启用：sudo ufw enable

允许端口：sudo ufw allow 22 (允许 SSH) 或 sudo ufw allow 80/tcp

拒绝连接：ufw deny 23         # 拒绝Telnet

**使用 firewall**
firewall-cmd --state
firewall-cmd --list-all

停止服务
systemctl stop firewalld
systemctl disable firewalld

- # 2.  对象存储是什么 对象存储和文件的关系是什么 有哪些桶设置的方式

对象存储是一种用于管理海量非结构化数据的现代数据存储架构。

它与传统的文件存储方式有本质区别。简单来说，它*不再使用文件夹和子文件夹的层级结构来组织数据*，而是将所有数据（如照片、视频、音频、文档等）都视为一个个独立的“对象”（Object），并将它们扁平化地存放在一个巨大的“桶”（Buckket)中

**🗂️ 对象存储与文件的关系**

你可以将对象存储中的概念与文件存储进行类比，以便更好地理解：
对象 (Object)：相当于文件。它是存储的基本单元，不仅包含文件数据本身，还附带了丰富的元数据（如创建时间、内容类型、自定义标签等）和一个全局唯一的标识符。

桶 (Bucket)：相当于一个顶级容器。你可以把它想象成一个根目录，但它内部是扁平的，没有多层级的文件夹结构。所有的对象都直接存放在桶里。

**1. 基础 ACL（访问控制列表）**
这是最简单粗暴的。直接在桶的 ACL 设置里，把“公共访问”或者“所有人（Everyone/All Users）”的权限勾选为 Read（读取）。

效果：只要知道文件的 URL，谁都能下载。

**2. Bucket Policy（桶策略）—— 最推荐，最专业**
这种方式最灵活。它是用一段 JSON 脚本来定义的。比如我想让 static 目录下所有的图片都能被公网访问，我会写个 Policy：

Effect: Allow

Principal: *（代表所有人）

Action: s3:GetObject

Resource: arn:aws:s3:::my-bucket/*

**3. 静态网站托管（Static Website Hosting）**
如果用户是为了刷网页用的，开启这个功能后，桶会自动变成一个 HTTP 服务器。

特点：它会强制要求桶里的内容具备公共读权限，否则网页打不开。
